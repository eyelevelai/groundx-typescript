# Architecture Notes — groundx-typescript

> Onboarding lens: official GroundX **TypeScript SDK**, the twin of `groundx-python`.
> Both are generated by **Fern** from the upstream OpenAPI spec in `eyelevel-fern-config`.
> Focus: the generated-vs-hand-written boundary, the editable layer, build/test gates,
> and the agent's editable perimeter. **Do NOT propose changes to generated code.**

_Onboarding complete — Phases 1–4. Deep dives: src/Wrapper.ts, src/core/fetcher._

## Agent editable perimeter (TL;DR for making changes)
- **Edit by hand:** `src/Wrapper.ts` (logic), `src/index.ts` (exports), `README.md`. New convenience
  methods/files must also be added to `.fernignore` or they'll be clobbered.
- **Never edit (regenerated):** `Client.ts`, `BaseClient.ts`, everything in `src/api/**` and
  `src/core/**`, `environments.ts`, `src/auth/**`, generated tests.
- **API surface changes** (new endpoints, request/response shapes) → upstream `eyelevel-fern-config`.
- **No CI safety net for `ingest()`** — add a test in `tests/custom.test.ts` (already .fernignore'd)
  if you touch the wrapper.

## System summary
Client library wrapping the GroundX REST API (document ingestion, search/RAG, buckets,
groups, workflows). Published to npm as `groundx` (v3.5.8). Dual-published CJS + ESM.
Most of the code is Fern-generated; a thin hand-written layer adds an `ingest()`
convenience method on top.

## Tech stack
- TypeScript ~5.9, dual CJS+ESM output
- Node 18+ (also Vercel, Cloudflare Workers, Deno, Bun, React Native); node-fetch fallback
- pnpm 10.33 (workspace), Biome 2.4 (lint+format), Vitest 4 + MSW (mock-server tests)
- Codegen: Fern CLI 5.37.9, generator `fern-typescript-node-sdk` 3.69.0
- Runtime deps: only `url-join`

## The generated-vs-hand-written boundary (load-bearing)
`.fernignore` is the contract — the ONLY files Fern won't overwrite:
```
src/Wrapper.ts
src/index.ts
README.md
```
- Safe to hand-edit: Wrapper.ts, index.ts, README.md.
- Off-limits (regenerated, clobbered on next `fern generate`): Client.ts, BaseClient.ts,
  everything in api/ and core/, environments, auth, tests.
- API-surface changes belong UPSTREAM in `eyelevel-fern-config`, not here.
- `.fern/metadata.json` pins the origin spec commit.

## Core data flow — `client.ingest(documents)`
1. `Wrapper.ts:16` validate + partition docs by filePath:
   - isUrl → remoteDocuments[]; isValidLocalPath (fs.statSync) → localDocuments[]; else throw.
2. `processLocalDocuments` (:157) — serial per file:
   2a. `getPresignedUrl` (:65) GET {uploadApi}?name=&type= via core.fetcher, auth via
       `_getCustomAuthorizationHeaders()` (X-API-Key, NOT the AuthProvider).
   2b. PUT bytes to presigned URL via RAW global fetch (:140) — no retries/timeout/auth/logging.
   2c. derive sourceUrl (GX-HOSTED-URL header or stripped URL) + fileType (ext→MIME→DocumentType).
3. push uploaded onto remoteDocuments (everything now "remote").
4. `this.documents.ingestRemote({documents, callbackUrl, callbackData})` [GENERATED]
   → __ingestRemote (Client.ts:115): authProvider.getAuthRequest() → mergeHeaders →
     POST {baseUrl|environment|Default}/v1/ingest/documents/remote via (options.fetcher ?? core.fetcher)
     → requestWithRetries (408/429/5xx, exp backoff + jitter, honors Retry-After/X-RateLimit-Reset)
     → ok: IngestResponse; 400: BadRequestError; 401: UnauthorizedError; else GroundXError.

Upload service host (`api.eyelevel.ai/upload/file`) is separate from the API base URL.

## Key abstractions
1. **AuthProvider / Supplier** — pluggable auth (HeaderAuthProvider→X-API-Key; NoOpAuthProvider for
   auth:false). `Supplier<T>` = value or async fn returning it; resolved via `core.Supplier.get()`.
2. **HttpResponsePromise + WithRawResponse** — await for body, `.asRaw()` for headers+body.
   Uniform `public method()` wraps `private __method()` pattern across every resource.
3. **Centralized fetch pipeline** (core.fetcher → requestWithRetries) — one shared retry/backoff/
   timeout/error-classification impl. Route through a resource client to inherit it.

## Per-module notes

### src/Wrapper.ts (HAND-WRITTEN — the entire editable logic surface; 282 lines)
- Public surface = exactly one method: `ingest(documents, callbackUrl?, callbackData?, requestOptions?, uploadApi="https://api.eyelevel.ai/upload/file")`.
- Private helpers: processLocalDocuments, uploadFile, getPresignedUrl, expandedPath,
  getFileExtension + 3 type maps (DOCUMENT_TYPE_ALIASES, DOCUMENT_TYPE_TO_MIME, MIME_TO_DOCUMENT_TYPE),
  isUrl, isValidLocalPath, getFileName, readFile, _getCustomAuthorizationHeaders.
- Gotchas (verified):
  1. **ZERO test coverage** — no test imports Wrapper or calls .ingest(); wire tests import the
     generated `src/Client`; custom.test.ts is a placeholder. Regressions won't be caught by CI.
  2. **Two parallel auth paths** — generated uses AuthProvider; wrapper upload uses its own
     `_getCustomAuthorizationHeaders()`. Both emit X-API-Key; must change both.
  3. **Byte upload (2b) unguarded** — global fetch, no retries/timeout/custom-fetcher/logging.
  4. Header casing forced upper-case; only PUT accepted (throws otherwise).
  5. `md` silently ingested as `txt` (alias).
  6. MIME_TO_DOCUMENT_TYPE derived by inverting the forward map — edit one, both change.
- Divergence from groundx-python twin: Python's hand layer (ingest.py + csv_splitter.py + extract/
  agent subsystem) adds batching (MAX_BATCH_SIZE=50, 50MB), ingest_directory(), CSV splitting,
  extract subsystem, and async variants. **TS has NONE of these** — single unbatched serial ingest().
  Feature-parity candidates if porting from Python: batching + directory ingest, added to this file.

### src/core/fetcher (GENERATED — shared request pipeline; do NOT edit, regenerated)
- Entry: `core.fetcher` = `fetcherImpl` (Fetcher.ts:413/257). Used by every generated __method and
  by Wrapper.getPresignedUrl. The wrapper's byte-upload (uploadFile PUT) is the only path that skips it.
- Lifecycle: build URL (queryString | createRequestUrl) → getRequestBody(by type) → getFetchFn() →
  build headers (Accept by responseType + Content-Type + per-header EndpointSupplier) → debug log
  (redacted) → requestWithRetries(makeRequest) → classify 200–399 ok / else status-code error → catch.
- Timeout/abort (makeRequest.ts + signals.ts): getTimeoutSignal(AbortController+setTimeout) fused with
  user abortSignal via anySignal; either aborts fetch; timeout cleared on completion. Catch checks
  abortSignal.aborted first (→ "unknown"/"user aborted"), else error.name==="AbortError" (→ "timeout").
- Gotchas (verified):
  1. **Retries are HTTP-status-only.** requestWithRetries retries only 408/429/5xx *responses*; if
     makeRequest THROWS (network/timeout/DNS) the error propagates with NO retry.
  2. **No response validation** — Fern config noSerdeLayer:true + skipResponseValidation:true, so
     success bodies are cast (`as R`), never schema-checked.
  3. **getFetchFn() just returns global `fetch`** (3 lines). README's "defaults to node-fetch" is STALE
     for this generation (relies on Node 18+ global). Custom fetch via fetch/fetcher options still works.
  4. **Log redaction** scrubs large SENSITIVE_* sets (x-api-key, tokens, cookies, URL userinfo,
     percent-encoded keys). Safe to debug-log; but only KNOWN sensitive names are masked.
  5. Error reasons `non-json` / `body-is-null` are declared but produced in getResponseBody/
     getErrorResponseBody, not the main fetcherImpl path (why Wrapper.getPresignedUrl branches on them).
  6. `cache:"no-store"` feature-detected per runtime (CF Workers throw); applied only for streaming/SSE.
- Error reasons → generated clients map "status-code" to typed BadRequestError/UnauthorizedError/GroundXError.
